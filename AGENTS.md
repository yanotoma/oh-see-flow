# oh-see-flow

Custom workflow configuration for opencode.

## Global Rules

- Always use Engram memory to persist important decisions and learnings
- Prefer editing existing files over creating new ones
- Follow existing code conventions in the project you're working on
- Never commit secrets, API keys, or credentials
- Run lint/typecheck commands after making changes when available

## Workflow Preferences

- Start with the brainstorming skill before creative work
- Use systematic debugging before proposing fixes
- Use test-driven development when implementing features
- Verify work before claiming completion

## Code Style

- Be concise in responses
- Avoid unnecessary comments in code unless asked
- Follow the existing patterns of the codebase you're modifying
